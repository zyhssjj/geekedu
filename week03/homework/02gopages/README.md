# 日志处理系统（生产者-消费者模型）

## 项目概述
本项目基于 Go 语言的 Goroutine 和 Channel 实现「生成日志 → 过滤日志 → 存储日志」的三阶段日志处理系统，严格遵循生产者-消费者设计模式，支持并发过滤、全局停止信号触发及所有 Goroutine 优雅退出，完全满足作业要求。

## 功能特性
1. **日志生成**：单 Goroutine 循环生成 100 条日志（ID 从 1 到 100），包含唯一 ID 和模拟业务内容；
2. **并发过滤**：3 个 Goroutine 并发消费日志，筛选偶数 ID 作为「重要日志」，奇数 ID 直接忽略；
3. **全局停止**：当检测到 ID=50 的日志时，触发全局停止信号，所有 Goroutine（生产者、过滤器、存储者）立即优雅退出；
4. **日志存储**：单 Goroutine 接收过滤后的重要日志，通过打印模拟存储到文件/数据库的过程；
5. **安全协调**：使用 `sync.WaitGroup` 协调过滤器 Goroutine，确保所有过滤任务完成或收到停止信号后，安全关闭通道，避免资源泄露。

## 核心设计
### 1. 数据流转架构
- **logChannel**：缓冲容量 10，用于传递生产者生成的原始日志，避免生产者因阻塞导致效率低下；
- **filteredChannel**：缓冲容量 5，用于传递过滤后的重要日志，解耦过滤器和存储者；
- **stopChan**：缓冲容量 1 的双向通道，用于传递全局停止信号，支持多 Goroutine 安全发送。

### 2. 核心组件说明
| 组件         | 数量 | 核心职责                                                                 | 关键逻辑                                                                 |
|--------------|------|--------------------------------------------------------------------------|--------------------------------------------------------------------------|
| 生产者       | 1个  | 生成 ID 1-100 的日志，发送到 logChannel；监听停止信号，收到后立即终止生成。 | 循环生成日志，通过 `select` 监听 stopChan，避免阻塞在通道发送。           |
| 过滤器       | 3个  | 并发消费 logChannel，筛选偶数 ID 日志到 filteredChannel；检测 ID=50 并触发停止。 | 用 `for range` 接收日志，`select` 监听停止信号；非阻塞发送停止信号，避免重复发送阻塞。 |
| 存储者       | 1个  | 消费 filteredChannel 中的重要日志，打印模拟存储；监听停止信号，收到后退出。 | 双重监听（停止信号 + 通道关闭），确保无日志时正常退出，有信号时立即退出。 |
| 协调器       | 1个  | 等待所有过滤器完成或收到停止信号，关闭 filteredChannel，避免存储者阻塞。   | 用「Goroutine + 临时通道」替代高版本 API，兼容所有 Go 版本。             |

### 3. 关键机制详解
#### （1）并发控制
- 过滤器通过 `sync.WaitGroup` 管理：启动前调用 `wg.Add(3)`，每个过滤器退出前调用 `wg.Done()`，确保主线程能感知所有过滤器状态；
- 通道缓冲设计：避免 Goroutine 因通道无接收者而永久阻塞，提高并发吞吐量。

#### （2）优雅退出
- 全局停止信号：`stopChan` 为缓冲通道（容量 1），支持任意 Goroutine 发送信号，且不会因重复发送导致阻塞；
- 全链路监听：生产者、过滤器、存储者均在关键流程中通过 `select` 监听 stopChan，收到信号后立即清理资源并退出，无死循环或资源泄露。

#### （3）通道安全关闭
- logChannel：生产者生成 100 条日志后或收到停止信号后关闭，告知过滤器无更多日志；
- filteredChannel：协调器在「所有过滤器完成」或「收到停止信号」后关闭，告知存储者无更多重要日志，避免存储者永久阻塞在通道接收。
